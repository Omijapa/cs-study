# Day 10 - 동기화 문제, 교착 상태(Dead Lock)

## 동기화

여러 프로세스 또는 스레드가 공유 자원에 동시에 접근할 때 데이터의 일관성을 유지하도록 실행 순서를 제어하는 것

### 주요 용어

- 경쟁 상태 (Race Condition) : 여러 프로세스/스레드가 동시에 공유 자원에 접근하여 실행 순서에 따라 결과가 달라지는 현상
- 임계 구역 (Critical Section) : 공유 자원에 접근하는 프로그램 내부의 코드 영역
- 상호 배제 (Mutual Exclusion) : 한 프로세스가 임계 구역에 있으면 다른 프로세스는 진입할 수 없음

### 임계 구역 문제 (Critical Section Problem)

임계 구역 문제를 해결하기 위해서는 다음 세 가지를 만족해야 한다.

| 요구 사항                        | 설명                                                                                                        |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| **상호 배제 (Mutual Exclusion)** | 한 프로세스가 임계 구역 실행 중이면, 다른 프로세스는 진입할 수 없어야 한다.                                 |
| **진행 (Progress)**              | 임계 구역에 진입하려는 프로세스가 있다면, 결정이 무한정 연기되지 않고 다음 진입 프로세스가 결정되어야 한다. |
| **유한 대기 (Bounded Waiting)**  | 임계 구역 진입을 요청한 프로세스는 무한정 기다리지 않고, 언젠가는 진입할 수 있어야 한다.                    |

### 동기화 해결 기법

1. 뮤텍스 (Mutex) : 락(열쇠)을 가진 오직 1개의 스레드만 공유 자원 사용 가능. (소유권 개념)
2. 세마 포어 (Semaphore) : 지정한 개수(N개)의 스레드가 동시에 자원 사용 가능 (접근 가능한 자원 개수를 나타내는 정수 변수와 이를 제어하는 P(wait), V(signal) 연산으로 구성됨)
3. 모니터 (Monitor): 프로그래밍 언어 차원에서 제공하는 고수준 동기화 도구 (ex. Java의 synchronized)

## 교착 상태 (Dead Lock)

두 개 이상의 프로세스가 서로가 가진 자원을 기다리며 무한 대기 상태에 빠지는 현상

```c
P1
Lock A 획득
Lock B 대기

P2
Lock B 획득
Lock A 대기

그 결과,
P1 ←→ P2
무한 대기
```

### 교착 상태 발생 조건

1. 상호 배제 : 자원은 한 번에 한 프로세스만 사용 가능
2. 점유와 대기 : 자원을 가진 상태에서 다른 자원을 기다림
3. 비선점 : 다른 프로세스의 자원을 강제로 뺏을 수 없음
4. 순환 대기 : 대기 프로세스들이 고리 형태로 서로를 기다림

| 요구 사항                        | 설명                                       |
| -------------------------------- | ------------------------------------------ |
| **상호 배제 (Mutual Exclusion)** | 자원을 동시에 사용할 수 없음               |
| **점유와 대기 (Hold and Wait)**  | 자원을 가진 상태에서 다른 자원을 요청함    |
| **비선점 (No Preemption)**       | 다른 프로세스의 자원을 강제로 뺏을 수 없음 |
| **순환 대기 (Circular Wait)**    | 대기 프로세스들이 원형으로 서로를 기다림   |

### 교착 상태 처리 방법

#### 1. Prevention (예방)

4가지 조건 중 하나를 제거한다. 예를 들어 Circular Wait을 제거하기 위해 항상 A → B 순서로만 Lock을 획득하게 한다.

#### 2. Avoidance (회피)

Deadlock이 발생할 가능성이 있는 상태에 진입을 금지한다. (Banker’s Algorithm : Safe State만 허용)

#### 3. 감지 및 복구 (Detection & Recovery)

Deadlock 발생은 허용하되, 이후 탐지되면 시스템에 문제가 발생하지 않도록 조치를 취한다.

#### 4. 무시 (Ignore)

현실적인 방법. 데드락 처리에는 오버헤드가 크기 때문에, 데드락이 매우 드물게 발생한다면 성능을 위해 무시하고, 예외 처리나 재부팅으로 해결한다.
