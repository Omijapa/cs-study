# Day 1-2 Bubble Sort 버블 정렬

## 개념 (WooVictory ver)
- 서로 인접한 두 원소를 검사하여 정렬하는 알고리즘이다.
- 인접한 2개의 원소를 비교해 크기가 순서대로 되어 있지 않으면 서로 교환한다.
- 선택 정렬과 기본 개념이 유사하다.

## 로직 (Python)
```python
def bubble_sort(arr):
    n = len(arr)

    for i in range(n):
        # 1회전이 끝날 때마다 가장 큰 값이 뒤로 이동하므로 이미 정렬된 뒤쪽 원소는 비교하지 않는다
        for j in range(0, n - i - 1):
            # 앞의 값이 뒤의 값보다 클 경우 교환
            if (arr[j] > arr[j+1]):
                arr[j], arr[j+1] = arr[j+1], arr[j]

        print(f"{i+1}단계: {arr}")

    return arr

arr = [7, 6, 2, 4, 3, 9, 1]
bubble_sort(arr)
```

버블 정렬은 정렬이 되어있건, 안되어있건 2개의 원소를 비교하기 때문에 최악의 경우, 최선의 경우, 평균의 경우 모두 시간 복잡도가 O(N^2)으로 동일

## 장점
- 구현이 간단하고 소스코드가 직관적이다.
- 개선된 버블 정렬*에서는 이미 정렬된 데이터의 경우 빠르게 종료할 수 있다.
- 정렬하고자 하는 배열 안에서 정렬하는 방식이므로, 다른 메모리 공간을 필요로 하지 않는다.
- 안정 정렬이다.

## 단점
- 시간 복잡도가 최악, 최선, 평균 모두 O(N^2)이므로 비효율적이다.
- 다른 정렬에 비해 정렬 속도가 느리다.
- 교환 횟수가 많다.
- 역순배열을 정렬할 때 가장 느리다.

## 참고
- for j in range(0, n - i - 1) 인 이유
0 ~ n-i까지 비교해야하니 range(0, n-i)라고 생각할 수 있지만, 다음 코드를 보면 arr[j]와 arr[j+1]를 비교하기 때문에 결국 0 ~ n-i까지 전부 비교하는 것이다.

- 개선된 버블 정렬*
swapped 같은 변수를 이용해서 교환이 한 번도 안 일어나면 종료하는 최적화 버전(교환이 한번도 안일어난 경우 이미 정렬된 상태)

```python
# 개선된 버블 정렬
def bubble_sort(arr):
    n = len(arr)

    for i in range(n):
        swapped = False

        # 1회전이 끝날 때마다 가장 큰 값이 뒤로 이동하므로
        # 이미 정렬된 뒤쪽 원소는 비교하지 않는다.
        for j in range(0, n - i - 1):
            # 앞의 값이 뒤의 값보다 클 경우 교환
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True

        print(f"{i + 1}단계: {arr}")

        # 이번 회전에서 교환이 한 번도 없었다면 이미 정렬된 상태
        if not swapped:
            break

    return arr


arr = [7, 6, 2, 4, 3, 9, 1]
bubble_sort(arr)

```
