# Day 1-1 Selection Sort 선택 정렬

## 개념 (WooVictory ver)
- 해당 순서에 원소를 넣을 위치는 이미 정해져있고, 어떤 원소를 넣을지 선택하는 알고리즘이다.
- 현재 위치에 저장될 값의 크기가 작냐, 크냐에 따라서 최소 선택 정렬(오름차순 정렬)과 최대 선택 정렬(내림차순   정렬)로 나뉜다.

## 개념 (hoon ver)
- 배열의 앞 인덱스부터 차례대로 자리를 확정하면서 아직 정렬되지 않은 구간에서 가장 작은 값을 찾아 그 자리에 넣는 정렬
- 첫 번째 인덱스부터 차례대로 확정하면서 정렬된 구간을 앞쪽부터 하나씩 늘려가는 느낌

## 로직 (Python)
```python
def selection_sort(arr):
    n = len(arr)

    for i in range(n):
        min_index = i

        # i 이후 구간에서 최솟값의 위치 찾기(i+1 ~ n)
        for j in range(i + 1, n):
            if arr[j] < arr[min_index]:
                min_index = j
        
        # 현재 위치 i와 최솟값 위치 min_index 교환
        arr[i], arr[min_index] = arr[min_index], arr[i]

    return arr

arr = [7, 6, 2, 4, 3, 9, 1]
selection_sort(arr)

print(arr)
```

- 간단하지만 매우 비효율적

1단계 => n-1번 비교
2단계 => n-2번 비교
3단계 => n-3번 비교
...
총 비교 횟수: n(n-1) / 2
시간복잡도: O(N^2)
공간복잡도: O(1)

## 장점
- 단순함
- 정렬하고자 하느 배열 안에서 교환 -> 다른 메모리 공간 불필요(공간복잡도 O(1))

## 단점
- 이미 정렬되어 있더라도 매 단계마다 남은 구간을 모두 비교해야 하므로 비효율적임
- 최선, 평균, 최악의 경우 모두 시간복잡도가 O(N^2)임
- 불안정 정렬(Unstable Sort)임
