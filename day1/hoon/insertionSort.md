# Day 1-4 Insertion Sort 삽입 정렬

## 개념 (WooVictory ver)
- 손 안의 카드를 정렬하는 방법과 유사하다.
- 새로운 카드를 기존의 정렬된 카드 사이에 올바른 자리를 찾아 삽입한다.
- 2번째 원소부터 시작하여 그 앞(왼쪽)의 원소들과 비교하여 삽입할 위치를 지정한 후, 원소를 뒤로 옮기고 지정된 자리에 자료를 삽입하여 정렬하는 알고리즘이다.
- 최선의 경우, O(N)이라는 엄청나게 빠른 효율성을 가지고 있어, 다른 정렬 알고리즘의 일부로 사용될 만큼 좋은 정렬 알고리즘이다.

## 로직 (Python)
```python

def insertion_sort(arr):
    n = len(arr)

    # 첫 번째 원소는 이미 정렬되어 있다 가정. 두 번째 원소부터 시작
    for i in range(1, n):
        standard = arr[i] # 삽입할 값을 standard에 넣음
        index = i - 1 # standard 바로 앞의 index

        # standard보다 큰 값들을 오른쪽으로 한 칸씩 이동
        while index >= 0 and arr[index] > standard:
            arr[index + 1] = arr[index]
            index -= 1
        
        # standard가 들어갈 위치에 삽입(비어있는 자리)
        arr[index + 1] = standard
    
    return arr

arr = [7, 6, 2, 4, 3, 9, 1]
insertion_sort(arr)

print(arr)

# 단계별 결과

5단계: [2, 3, 4, 6, 7, 9, 1]

###
```
- 이미 정렬된 경우(최선의 경우)
1단계: 1번 비교
2단계: 1번 비교
3단계: 1번 비교
...
n-1단계: 1번 비교
총 비교 횟수: n-1
시간복잡도: O(N)

- 역순으로 정렬된 경우(최악의 경우)
1단계: 1번 비교
2단계: 2번 비교
3단계: 3번 비교
...
n-1단계: n-1번 비교
총 비교 횟수: n(n-1) / 2
시간복잡도: O(N^2)

- 평균적으로
시간복잡도: O(N^2)


공간복잡도(새 배열을 만들지 않음): O(1)


## 장점
- 안정 정렬이다
- 대부분의 원소가 이미 정렬되어 있는 경우, 매우 효율적일 수 있다.
- 정렬하고자 하는 배열 안에서 교환하는 방식이므로, 다른 메모리 공간을 필요로 하지 않는다.
- 선택 정렬이나 버블 정렬에 비하여 상대적으로 빠르다.

## 단점
- 비교적 많은 수들의 이동을 포함한다.
- 비교해야 할 원소 수가 많고 배열의 크기가 클 경우 비효율적이다.
- 평균과 최악의 시간 복잡도가 O(N^2)이므로 비효율적이다.
