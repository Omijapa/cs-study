# Day 2-1 Quick Sort 퀵 정렬

## 개념 (WooVictory ver)
- 퀵 정렬은 분할 정복 방법을 통해 주어진 배열을 정렬한다.
- 분할 정복(Divide and Conquer) : 문제를 작은 2개의 문제로 분리하고 각각을 해결한 다음, 결과를 모아서 원래의 문제를 해결하는 전략이다.
- 불안정 정렬에 속하며, 다른 원소와의 비교만으로 정렬을 수행하는 비교 정렬에 속한다.
- 또한, Merge Sort와 달리 Quick Sort는 배열을 비균등하게 분할한다.
- 분할(Divide) : 주어진 배열을 피벗을 기준으로 비균등하게 2개의 부분 배열로 분할한다.(피벗을 중심으로 왼쪽 : 피벗보다 작은 요소들, 오른쪽 : 피벗보다 큰 요소들)
- 정복(Conquer) : 부분 배열을 정렬한다. 부분 배열의 크기가 충분히 작지 않으면 순환 호출을 이용해 다시 분할 정복 방법을 적용한다.

## 개념(알고리즘 흐름) (hoon ver)
- 1. pivot을 기준으로 분할
- 2. pivot의 왼쪽에는 pivot보다 작은 것들 / 오른쪽에는 pivot보다 큰 것들이 가도록 배치
- 3. pivot을 기준으로 작은 값들은 왼쪽 구간에, 큰 값들은 오른쪽 구간에 가깝게 배치한다. 이 두 배열에서 각각 pivot을 설정해 재귀적으로 작업
=> Divide and Conquer 방식

## 로직 (Python)
```python
def quick_sort(arr, left, right):
    L = left # 배열의 시작 인덱스
    R = right # 배열의 끝 인덱스
    pivot = arr[(left + right) // 2] # 현재 구간의 가운데 인덱스 값을 pivot으로 설정

    # 증가하는 L 인덱스가 감소하는 R 인덱스보다 커질 때 종료
    while L <= R:

        # L 인덱스에서 +하면서 pivot보다 큰 값 찾기
        # pivot보다 작은 값은 왼쪽에 있어도 되므로 넘어감.
        while arr[L] < pivot:
            L += 1

        # R 인덱스에서 -하면서 pivot보다 작은 값 찾기
        # pivot보다 큰 값은 오른쪽에 있어도 되므로 넘어감
        while arr[R] > pivot:
            R -= 1
        
        # L, R위치의 값을 교환하므로써 pivot보다 작은 값은 왼쪽에, pivot보다 큰 값은 오른쪽으로 가게 분할함
        if L <= R:
            arr[L], arr[R] = arr[R], arr[L]
            L += 1 # 다음 교환 대상을 찾기 위해 넘어감
            R -= 1 # 다음 교환 대상을 찾기 위해 넘어감
        
    # 왼쪽 부분 배열 정렬*
    if left < R:
        quick_sort(arr, left, R)
    
    # 오른쪽 부분 배열 정렬
    if L < right:
        quick_sort(arr, L, right)


arr = [5, 3, 8, 4, 9, 1, 6, 2, 7]

print("정렬 전:", arr)

quick_sort(arr, 0, len(arr) - 1)
```

## 시간복잡도
### 1. 최선의 경우: O(NlogN)
pivot을 기준으로 배열이 거의 절반씩 나누어지는 경우
```text
N
→ N/2, N/2
→ N/4, N/4, N/4, N/4
→ ...

1단계: N번 비교
2단계: N번 비교
3단계: N번 비교
...
log N 단계

```
총 비교 횟수: N x logN
최선의 시간복잡도: O(NlogN)

### 2. 평균의 경우: O(NlogN)
평균적으로도 pivot이 어느 정도 균형 있게 배열을 나누면 재귀 깊이가 log N에 가까워짐
평균 시간복잡도: O(NlogN)

### 3. 최악의 경우: O(N^2)
pivot이 계속 최솟값이나 최댓값으로 잡혀서 배열이 한쪽으로 치우쳐 나뉘는 경우
```text
N
→ 0개, N-1개
→ 0개, N-2개
→ 0개, N-3개
→ ...
재귀 깊이가 N에 가까워짐

비교 횟수 누적
N + (N-1) + (N-2) + ... + 1 = N(N+1) / 2
```
최악의 시간복잡도: O(N^2)

## 공간복잡도
- Merge Sort처럼 새로운 배열을 만들지는 않음
- 재귀 호출을 사용하기 때문에 재귀 호출 스택 공간 필요

### 1. 최선/평균 공간 복잡도
배열이 균형 있게 나뉘어졌을 때 재귀 깊이: logN
평균적인 공간 복잡도: O(logN)

### 2. 최악 공간 복잡도
배열이 한쪽으로만 계속 나뉘어졌을 때 재귀 깊이: N
최악의 공간 복잡도: O(N)

## 장점
- 불필요한 데이터의 이동을 줄이고 먼 거리의 데이터를 교환할 뿐만 아니라, 한번 결정된 피벗들이 추후 연산에서 제외되는 특성 때문에, 시간 복잡도가 O(N logN)을 가지는 다른 정렬 알고리즘과 비교했을 때도 가장 빠르다.
- 정렬하고자 하는 배열 안에서 교환하는 방식이므로, 다른 메모리 공간을 필요로 하지 않는다. 다만 재귀 호출을 사용하므로 호출 스택 공간은 필요하다.

## 단점
- 불안정 정렬이다.
- pivot 선택이 좋지 않아 계속 최솟값이나 최댓값이 pivot으로 선택되면 분할이 한쪽으로 치우쳐 O(N²)의 시간이 걸릴 수 있다.
- 특히 pivot을 맨 앞이나 맨 뒤 원소로 선택하는 구현에서는 이미 정렬된 배열이나 역정렬된 배열에서 최악의 경우가 발생하기 쉽다.

## 참고
- 재귀 정렬 부분에 대한 부가 설명
```python
    # 왼쪽 부분 배열 정렬
    if left < R:
        quick_sort(arr, left, R)
    
    # 오른쪽 부분 배열 정렬
    if L < right:
        quick_sort(arr, L, right)
```
```text
`while (L <= R)` 반복이 끝나면 L과 R은 서로 엇갈린 상태가 된다.


left                 R   L                 right
 |-------------------|   |-------------------|
 pivot보다 작거나 같은 쪽   pivot보다 크거나 같은 쪽

`left < R`은 왼쪽 구간에 원소가 2개 이상 남아 있는 경우이고,  
`L < right`는 오른쪽 구간에 원소가 2개 이상 남아 있는 경우이다.

 이렇기 때문에 재귀의 조건문 상 왼쪽, 오른쪽 배열에 각각 정렬할 원소가 2개 이상 남아 있으면 quick_sort를 다시 호출하는 시스템이다
```
