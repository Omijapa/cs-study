# Day 2-2 Heap Sort 힙 정렬

## 개념 (WooVictory ver)
- 완전 이진 트리를 기본으로 하는 힙 자료구조를 기반으로 한 정렬 방식이다.
- 불안정 정렬에 속한다.
1. 최대 힙을 구성한다.
2. 현재 힙 루트는 가장 큰 값이 존재한다. 루트의 값을 마지막 요소와 바꾼 후, 힙의 사이즈를 하나 줄인다.
3. 힙의 사이즈가 1보다 크면 위의 과정을 반복한다.

## 개념 (hoon ver)
- 완전 이진 트리를 기반으로 하는 정렬 알고리즘
- 배열을 Heap 자료구조로 만든 뒤, 가장 큰 값 또는 가장 작은 값을 하나씩 꺼내 정렬
- 오름차순 정렬 시 Max Heap을 일반적으로 사용함
- Max heap: 부모 노드의 값 >= 자식 노드의 값
- 힙 정렬은 비교 정렬에 속함
- 불안정 정렬임
- 배열 내부에서 원소 교환 -> 추가 배열 불필요
### 알고리즘 흐름
1. 주어진 배열을 Max heap으로 만든다
2. Max heap의 root에는 가장 큰 값이 위치
3. root와 배열의 마지막 원소를 교환함
4. 가장 큰 값(root에 있던 값)은 배열의 마지막 위치로 이동했으므로 정렬 완료 처리, 구간에서 제외한다
5. 남은 원소들에 대해 다시 max heap 만듦
6. 이 과정 반복 -> 배열 오름차순으로 정렬됨

## 로직 (Python)
```python
def heapify(arr, n, i): # arr: 배열, n: 배열 크기, i: 시작(root)
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2

    # 왼쪽 자식이 존재 & 현재 largest 보다 크면 새로 갱신
    if left < n and arr[left] > arr[largest]:
        largest = left
    
    # 오른쪽 자식이 존재 & 현재 largest보다 크면 새로 갱신
    if right < n and arr[right] > arr[largest]:
        largest = right
    
    # largest가 현재 노드가 아니라면, 부모와 자식의 위치 교환
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]

        # 교환 이후 아래쪽 힙 구조가 깨질 수 있으므로 다시 heapify
        heapify(arr, n, largest)

def heap_sort(arr):
    n = len(arr)

    # 1. Max heap 만들기
    # 마지막 부모 노드부터 루트 노드까지 heapify 수행
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)

    # 2. 하나씩 최대값을 뒤로 보내기
    for end in range(n -1, 0, -1):
        # root 노드의 값과 현재 힙의 마지막 값 교환
        arr[0], arr[end] = arr[end], arr[0]

        # 마지막 원소는 정렬 완료되었으므로 제외하고 다시 heapify
        heapify(arr, end, 0)

arr = [5, 3, 8, 4, 9, 1, 6, 2, 7]
print("정렬 전:", arr)
heap_sort(arr)
print("정렬 후:", arr)
```

## 시간복잡도
### 최대 힙 생성
배열 전체를 max heap으로 만드는 과정
```python
for i in range(n // 2 - 1, -1, -1):
    heapify(arr, n, i)
```
Max heap 생성 시간: O(N)

### 정렬 과정
root와 마지막 원소 교환
```python
for end in range(n-1, 0, -1):
    arr[0], arr[end] = arr[end], arr[0]
    heapify(arr, end, 0)
```
이 과정 N-1번 반복됨
heapify를 한번 수행할 때마다 힙의 높이만큼 내려감
=> 힙은 완전 이진 트리 => 높이: logN
정렬 과정의 시간복잡도: N번 반복 x heapify 비용(logN) = O(NlogN)

### 전체 시간복잡도
최대 힙 생성: O(N)
정렬 과정: O(NlogN)
전체 시간복잡도: O(NlogN)
- Heap Sort는 입력 배열의 정렬 상태와 관계없이 항상 힙을 만들고, 최댓값을 하나씩 꺼내는 과정을 반복한다. => 최선, 평균, 최악의 시간복잡도가 모두 같다.

## 공간복잡도
- Merge Sort처럼 별도의 큰 추가 배열을 만들지 않는다.
- 일반적인 Heap Sort의 공간복잡도는 O(1)임(재귀 구현 x)
- heapify()는 재귀 호출을 사용한다.
=> 재귀 호출 스택까지 고려하면 힙의 높이만큼 공간이 필요
공간복잡도: O(logN)



## 장점
- 최선, 평균, 최악의 시간복잡도가 모두 O(NlogN)으로 일정하다.
- 배열 내부에서 원소를 교환하므로 별도의 큰 추가 배열이 필요하지 않다.
- Quick Sort와 달리 최악의 경우에도 O(N²)로 떨어지지 않는다.

## 단점
- 불안정 정렬이다.
- 같은 O(NlogN) 정렬인 Quick Sort에 비해 실제 실행 속도는 느릴 수 있다.
- 힙 구조를 유지하기 위해 원소를 계속 비교하고 교환해야 하므로 캐시 효율이 좋지 않은 편이다.

## 참고
### 배열에서 힙의 인덱스 관계
힙은 트리 구조이지만, 실제 구현에서는 배열을 사용함
- 특정 노드의 인덱스를 i라고 가정
```text
  왼쪽 자식 인덱스: 2 * i + 1
  오른쪽 자식 인덱스: 2 * i + 2
  부모 인덱스: (i - 1 ) // 2
```
### heapify의 역할(gpt)
heapify는 특정 노드를 기준으로 아래쪽으로 내려가며 힙 조건을 만족하도록 만드는 함수이다.

최대 힙에서는 부모 노드가 자식 노드보다 크거나 같아야 한다.

부모 노드 >= 자식 노드

예를 들어 다음 배열이 있다고 하자.

[4, 7, 8, 5, 3, 1, 6, 2]

루트 노드 4는 자식 노드 7, 8보다 작다.

        4
      /   \
     7     8

최대 힙 조건을 만족하지 않으므로 가장 큰 자식인 8과 교환한다.

[8, 7, 4, 5, 3, 1, 6, 2]

하지만 교환된 4의 아래쪽에서도 힙 조건이 깨질 수 있다.

그래서 heapify()를 다시 호출해 아래쪽까지 힙 조건을 맞춘다.
