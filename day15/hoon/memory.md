# day15-2 Memory

## 컴퓨터 메모리
- `CPU가 프로그램을 실행하기 위해 필요한 명령어와 데이터를 저장하는 공간`
- 속도와 용량에 따라 계층적으로 구성됨
```text
    레지스터 -> 캐시 -> RAM -> SSD/HDD
```

CPU에 가까울수록 속도는 빠르지만 용량이 작고 가격이 비쌈
- `레지스터`: CPU 내부에 있으며 계산 중인 값이나 명령어를 저장함
- `캐시 메모리`: CPU가 자주 사용하는 데이터를 미리 저장하여 RAM 접근 시간을 줄임
- `RAM`: 실행 중인 프로그램과 데이터를 저장하는 주기억장치, 전원이 꺼지면 데이터가 사라짐
- `SSD/HDD`: 프로그램과 파일을 장기간 저장하는 보조기억장치

### 프로세스의 메모리 영역
- `Code 영역`: 실행할 프로그램의 명령어 저장
- `Data 영역`: 전역 변수와 정적 변수 저장
- `Heap 영역`: 실행 중 동적으로 생성된 객체 저장
- `Stack 영역`: 지역 변수, 매개변수, 함수 호출 정보 저장

## 참고
- 운영체제는 가상 메모리를 사용하여 각 프로세스에 독립적인 주소 공간을 제공
- 가상 주소는 페이지 테이블을 통해 실제 RAM의 물리 주소로 변환
- 접근하려는 페이지가 RAM에 없으면 페이지 폴트가 발생하며, 운영체제가 해당 페이지를 디스크에서 RAM으로 가져옴

```text
면접 답변 예시(컴퓨터 메모리란?)

컴퓨터 메모리는 CPU가 프로그램을 실행하는 데 필요한 명령어와 데이터를 저장하는 공간입니다. 메모리는 레지스터, 캐시, RAM, 보조기억장치 순으로 계층화되어 있으며, CPU에 가까울수록 빠르지만 용량이 작습니다. 또한 프로세스의 메모리는 Code, Data, Heap, Stack 영역으로 구분됩니다. 운영체제는 가상 메모리를 통해 프로세스별 독립적인 주소 공간을 제공하고, 페이지 테이블을 이용해 가상 주소를 물리 주소로 변환합니다.
```
