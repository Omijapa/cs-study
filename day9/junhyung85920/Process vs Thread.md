# 개념

<aside>
📜

**Process란?**

디스크에 저장되어 있던 **프로그램**이 메모리에 올라와 OS로부터 독립된 메모리 공간을 할당받고 실행 중인 상태 또는 실행 단위

특징

- 각 프로세스는 고유한 메모리 영역(Code, Data, Heap, Stack)을 가진다.
    - Code: 소스코드
    - Data: Global, Static 변수 등 프로그램 실행 동안 계속 유지될 데이터
    - Heap: 필요 시 동적 할당되는 메모리 공간
    - Stack: 함수 호출 시 사용하는 Local, Param, Retrun Address 등 임시 공간

**Thread란?**

프로세스가 할당받은 자원을 이용해 실제로 작업을 수행하는 최소 실행 단위

특징

- 하나의 프로세스 안에는 최소 하나 이상의 스레드가 존재한다.
- 같은 프로세스에 대한 스레드 간에 Code, Data, Heap 영역을 공유하며, 고유의 Stack 영역을 가진다.
    - 왜 stack영역만 별도로 할당받는가?
    → thread가 독립적인 실행 흐름이 되려면, 각 thread가 어떤 함수를 호출했고 어디까지 실행했는지 독립적으로 기록할 수 있어야 하기 때문이당
</aside>

# Multi-Process

하나의 프로그램을 여러 개의 프로세스로 구성하여 실행하는 것

- 독립된 구조라 하나의 프로세스가 죽어도 다른 프로세스에 영향 X
- 새로운 프로세스 생성 시 메모리를 통째로 할당 해야하고, Context Switching 할 때 속도가 느림

---

# Multi-Thread

하나의 프로세스 안에 여러 개의 스레드를 두어 작업을 동시에 처리하는 것

- 하나의 프로세스가 할당 받은 메모리 영역 중 일부를 공유하기 때문에, 훨씬 가볍고 빠르다.
- 공유된 메모리 영역이 오염될 가능성이 있음.
- Race Condition
