# 🍒 Omijapa CS Study

## 진행 방식

1. 각자 학습한 내용을 정리하여 GitHub에 업로드합니다.
2. 브랜치는 `dayN-name` 형식으로 생성합니다.
   - 예시: `day1-ssong`

3. 커밋 메시지는 자유롭게 작성합니다.
   - 예시: `Day 1 제출 (ssong)`

4. 평일(주 5일) 매일 20:00까지 PR(Pull Request)을 생성합니다.
5. 학습 분량은 주제의 난이도와 범위를 고려하여 스터디원들과 함께 결정합니다.
6. 마감 시간 이후 제출 시 벌금 1,000원이 부과됩니다.
7. 이 스터디의 끝에는 CS 지식으로 숙성된 새로 오미자가 있습니다. 🍶
