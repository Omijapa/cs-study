# Day 3 투 포인터 알고리즘

## 1. 투 포인터란?
- 1차원 배열에서 두 개의 포인터를 사용해 구간이나 두 원소를 탐색하는 알고리즘
- 보통 start, end 두 포인터를 사용
- 두 포인터를 한 방향으로 움직이면서 원하는 조건을 만족하는 값을 찾음

## 2. 왜 사용하는가?
- 완전 탐색으로 모든 구간 확인 -> 시간 초과 가능성 높음(배열 길이가 N일 때 모든 부분 배열 확인 시 O(N^2))
- 투 포인터 사용 시 start와 end가 각각 최대 N번 이동 -> 전체 시간 복잡도 O(N)으로 줄일 수 있음

## 3. 사용할 수 있는 조건(연속 부분합 문제의 경우)
- 배열의 원소가 양수이어야 함
    - end의 인덱스가 증가할 경우 합이 증가해야함
    - start의 인덱스가 증가할 경우 합이 감소해야함
    => 포인터 이동에 따라 합이 예측 가능하게 변해야 한다.

## 4. 로직 방향
- 구간을 [start,end)라 생각 (start는 포함, end는 미포함)
- 구해야하는 부분합 M
- 현재 구간 합을 sum이라 하였을 때,
    sum < M : end의 인덱스를 증가시켜 구간을 넓힘
    sum >= M: start의 인덱스를 증가시켜 구간을 좁힘
    sum == M: 정답 개수 증가

## 6. 시간복잡도
- start와 end가 최대 N번 이동
- start 최대 N번, end 최대 N번 이동하므로 최대 이동 횟수는 2N
- 시간복잡도: O(N)

## 7. 공간복잡도
- 배열 크기 N이 커져도 추가로 필요한 변수 개수는 거의 변하지 않으므로 상수 공간
- 기본 투 포인터 자체의 공간 복잡도: O(1)
