HW 핵심 4대 요소

1. CPU
2. 주기억장치(RAM / Memory)
3. 보조기억장치 (Storage / HDD, SSD)
4. 입출력장치 (I/O Devices - 키보드, 스피커, 모니터 등)

각 요소에 대해 자세히 설명하려면 끝도 없으니 일단 동작 예시를 살펴보자

# 예시

#### 사용자가 SSD에 설치된 프로그램을 실행한다.

1. 실행 요청
    1. 사용자가 프로그램 아이콘을 클릭하면 OS에 프로그램 실행 요청이 전달된다.
2. 프로그램 적재
    1. OS는 보조기억장치에 저장된 실행 파일을 주기억장치로 가져온다.
3. 프로세스 생성
    1. 실행 중인 프로그램을 관리하기 위해 프로세스를 생성한다.
        1. 메모리 할당
        2. 프로세스 식별 번호 부여
        3. CPU 실행 정보
        4. 파일 및 입출력 자원
        5. 접근 권한
4. CPU 할당
    1. CPU Scheduler가 실행할 프로세스 선택 후 CPU를 할당한다.
5. 명령어 실행
    1. CPU는 메로리에서 프로그램의 명령어를 하나씩 가져와 실행한다.
6. 입출력 요청
    1. 파일을 읽거나 출력 시 OS에 해당 작업을 요청한다.
7. 프로그램 종료
    1. OS가 프로세스에 할당된 메모리와 자원을 회수한다.

```
프로그램 실행 요청
        ↓
SSD에서 프로그램 확인
        ↓
RAM으로 코드와 데이터 적재
        ↓
프로세스 생성
        ↓
CPU가 명령어 실행
        ↓
입출력 및 결과 처리
        ↓
프로세스 종료 및 자원 회수
```

---

OS가 프로세스를 관리하거나, 입출력을 다룰 때에 인터럽트, 폴링, DMA라는 개념을 알아야 하지만 일단은 여기까지..
