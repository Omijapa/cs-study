# day 8-2 컴퓨터 시스템의 동작 원리
## 1. 컴퓨터 시스템의 구성

컴퓨터 시스템은 CPU, 메모리, 입출력 장치, 장치 컨트롤러로 구성됨

- `CPU`: 프로그램의 명령어를 실행함
- `메모리`: 실행 중인 프로그램과 데이터를 저장함
- `장치 컨트롤러`: 각 입출력 장치를 제어함
- `로컬 버퍼`: 장치와 주고받는 데이터를 임시로 저장함

## 2. CPU와 입출력 장치의 동작

CPU와 입출력 장치는 동시에 작업할 수 있음
(ex) CPU: 프로그램 B 실행 / 디스크 컨트롤러: 프로그램 A의 입출력 처리
```text
[일반적인 입출력 과정]
1. 프로세스가 시스템 콜로 입출력을 요청
2. 운영체제가 장치 드라이버를 통해 장치 컨트롤러에 작업을 지시ㄴ
3. 장치가 입출력을 수행
4. 작업 완료 후 컨트롤러가 인터럽트를 발생
5. 운영체제가 입출력 완료를 처리
```
## 3. 인터럽트
* 앞에서 다룬 인터럽트 개념을 입출력 과정과 연결해 다시 정리함

`인터럽트`: 장치나 특정 사건이 CPU에 처리가 필요함을 알리는 신호
```text
[인터럽트 처리 과정]

1. CPU가 프로그램을 실행함
2. 인터럽트가 발생함
3. 현재 실행 문맥을 저장함
4. 인터럽트 처리 루틴을 실행함
5. 저장한 문맥을 복원함
6. 중단했던 프로그램을 다시 실행함
```

`인터럽트 벡터`: 인터럽트 종류와 처리 코드의 위치를 연결하는 자료구조
`인터럽트 서비스 루틴(ISR)`: 실제 인터럽트를 처리하는 커널 코드

## 4. 동기식 / 비동기식 입출력

### 동기식 입출력
입출력을 요청한 프로세스가 작업 완료까지 기다리는 방식
입출력을 기다리는 동안 해당 프로세스는 봉쇄 상태가 되지만, CPU는 다른 프로세스를 실행할 수 있음

### 비동기식 입출력
입출력을 요청한 뒤 기다리지 않고 다른 작업을 계속 수행하는 방식

## 5. DMA

`DMA(Direct Memory Access)`: CPU 대신 입출력 장치와 메모리 사이의 데이터 전송을 담당하는 방식
```text
CPU가 DMA에 전송 정보 설정 
            ↓ 
DMA가 장치와 메모리 사이 데이터 전송 
            ↓ 
완료 후 CPU에 인터럽트 발생
```
### 장점
- CPU의 데이터 복사 부담 감소
- 인터럽트 횟수 감소
- 대용량 입출력 성능 향상

## 6. 사용자 모드와 커널 모드

### 사용자 모드
- 일반 응용 프로그램이 실행되는 모드
- 하드웨어를 직접 제어할 수 없음
- 특권 명령을 실행할 수 없음

### 커널 모드
- 운영체제가 실행되는 모드
- 메모리, 입출력 장치 등 시스템 자원에 접근 가능
- 특권 명령 실행 가능

사용자 프로그램이 입출력이나 파일 작업이 필요하면 `시스템 콜`을 통해 운영체제에 요청함
```text
사용자 프로그램
    ↓ 시스템 콜
커널 모드
    ↓ 작업 처리
사용자 모드로 복귀
```

## 참고
### CPU 보호
운영체제는 `타이머`를 이용해 하나의 프로세스가 CPU를 계속 독점하지 못하도록 함
