# Page 교체 알고리즘

## Optimal

- 앞으로 가장 오랫동안 사용되지 않을 page를 쫓아냄
- 실현 불가능함, 개념적으로만 남아있음

## FIFO

- RAM에 가장 먼저 들어온 page를 쫓아냄

## LRU

- 최근에 사용하지 않은 page를 가장 먼저 쫓아냄
- 최근에 사용하지 않았다면 앞으로도 사용하지 않을 확률이 높다는 사실에서 나온 알고리즘

## LFU

- 사용 횟수가 가장 적은 page를 쫓아냄
- 프로그램 구동 초반에 집중적으로 쓰이다가 나중에 안쓰는 page가 있다면, 계속 RAM에 상주할 수 있다.

## Clock Algorithm

- 이것도 최근에 사용하지 않은 page를 쫓아내는 방식이지만, LRU에서 참조 시각을 기록하는 오버헤드를 줄이기 위한 기법이다.
- 각 page에 reference bit을 둔다.
- 시계 바늘이 회전하듯 page들을 순회하며, reference bit가 1이면 0으로 바꾸고 기회를 한 번 더 주며 넘어간다.
- reference bit가 0인 page를 만나면 바로 쫓아낸다.
- bit(dirty bit)를 하나 더 사용하여 수정 여부까지 고려하여 쫓아내는 방법도 있다.
