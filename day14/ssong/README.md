# Day 14 - 페이지 교체 알고리즘, 단편화

# Page Fault

프로세스가 필요한 페이지가 메모리에 올라와 있지 않아 운영체제가 이를 가상 메모리에서 찾아 올리는 현상이다. 메모리 공간 확보를 위해 기존에 올라가있던 페이지 중 하나를 제거해야하는데 이걸 결정하는 알고리즘이 페이지 교체 알고리즘이다.

# 페이지 교체 알고리즘

### 1. FIFO (First-In First-Out)

- 가장 먼저 메모리에 들어온 페이지를 교체하는 알고리즘
- 큐 형태로 구현
- 장점 : 구현이 쉽고 오버헤드가 적다
- 단점 : 자주 사용되는 페이지도 오래 있으면 제거된다. [벨라디 이상현상](https://en.wikipedia.org/wiki/B%C3%A9l%C3%A1dy%27s_anomaly)이 발생한다. (프레임 수를 늘렸는데 page fault가 증가)

### 2. Optimal (OPT)

- 앞으로 가장 오랫동안 사용되지 않을 페이지를 교체
- 즉, 다음에 가장 늦게 사용될 페이지를 제거
- 장점 : Page Fault가 가장 적다
- 단점: 미래의 참조를 알아야 하므로 실제로 구현은 불가능함

### 3. LRU (Least Recently Used)

- 가장 오랫동안 사용되지 않은 페이지를 교체
- 장점: 실제 사용 패턴과 잘 맞는다. Optimal 알고리즘에 가장 가까운 성능을 보인다.
- 단점 : 최근 사용 시간을 기록해야하고 구현 비용이 높다

### 4. LFU (Least Frequently Used)

- 가장 적게 사용된 페이지를 교체
- 페이지 접근 횟수를 기록하는 추가 공간이 필요함
- 장점 : 많이 사용하는 페이지를 유지할 수 있다
- 단점 : 오래 전에 많이 사용된 페이지가 계속 남아 있을 수 있고, 최근에 자주 사용되는 페이지가 계속 교체될 수 있다.

### 5. Clock

- second chance algorithm으로도 불린다
- 각 페이지마다 reference bit(R 비트)을 둔다. 페이지 교체가 필요할 경우 Clock Hand가 원형으로 페이지들을 검사한다. 대상이 되었을 때 R bit가 0이면 최근에 사용되지 않았으니 제거, 1이면 최근에 사용된 페이지이므로 R 비트를 0으로 바꾸고 다음 페이지를 검사한다.
- LRU와 비슷하지만, 교체되는 페이지의 참조 시점이 가장 오래되었음을 보장하지 않는다.

# 단편화

### 개념

프로세스의 생성과 종료가 반복되면서 메모리가 잘게 나뉘거나, 최소 할당 단위 때문에 사용되지 못하는 공간이 생기는 현상을 말한다. 즉, 메모리에 빈 공간이 충분히 있음에도 효율적으로 사용할 수 없는 상태이다.

### 외부 단편화 (External Fragmentation)

- 할당 가능한 메모리 공간은 충분하지만, 빈 공간이 여러 곳에 흩어져 있어 연속된 공간을 확보하지 못하는 현상이다.
- Compaction(압축)이나 Paging(페이징)을 통해 해결할 수 있다.

#### Dynamic Storage Allocation

- 외부 단편화 발생 시 빈 공간 중 어떤 공간에 할당할 지 정해야함
- First Fit : 처음으로 충분히 큰 공간에 할당 (빠르고, 가장 많이 쓰임)
- Best Fit : 가장 작은 공간에 할당 (남는 공간이 가장 적음)
- Worst Fit : 가장 큰 공간에 할당 (큰 공간을 계속 유지하려는 목적, 실제 성능은 좋지 않음)

### 내부 단편화 (Internal Fragmentation)

- 메모리를 할당 받았지만 할당된 영역 내부에 사용하지 않는 공간이 남는 현상
- 고정 분할(Fixed Partitioning)이나 Paging 처럼 미리 정해진 크기로 메모리를 할당하는 방식에서 나타난다
