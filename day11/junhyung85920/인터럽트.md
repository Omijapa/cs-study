**Interrupt란?**

CPU가 프로그램을 실행하고 있을 때, I/O나 시스템 내부에서 예기치 못한 이벤트가 발생하여 CPU에 신호를 보내 처리를 요청하는 방식

Interrupt가 없다면, CPU는 polling방식으로 다른 요소들에게 자원이 필요한지 물어봐야 하는데, 매우 비효율적이므로, Interrupt가 현대 컴퓨터의 표준으로 자리잡았당

</aside>

---

# 종류

- Hardware Interrupt
    - CPU 외부 장치의 하드웨어에서 보내는 신호로서,  I/O Interrupt, Timer Interrupt가 있다.
- Software Interrupt ( = Trap)
    - CPU 내부에서 발생하며, 특정 Instruction 실행에 의배 발생
    - System call, Exception

---

# 처리 과정

1. 현재 상태 백업
    1. 현재 실행 중이던 프로세스의 Context를 메모리에 저장
2. Interrupt Vector 확인
    1. 발생한 Interrupt 번호를 보고, 이 Interrupt를 처리할 코드의 주소가 담긴 Interrupt Vector를 조회
3. ISR (Interrupt Service Routine) 실행
    1. 해당 번호에 매핑된 ISR을 실행하여 문제 해결
        1. Interrupt 종류에 따라 pre-mapping된 Interrupt 처리방식을 조회 후 실행한다.
4. 하던 일 재개
    1. 백업해 둔 Context 복원 후 진행
