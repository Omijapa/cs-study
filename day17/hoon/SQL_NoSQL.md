# day17-1 SQL vs NoSQL

## 1. SQL 데이터베이스
- 데이터를 `테이블 형태`로 저장하는 관계형 데이터베이스
- 대표적인 DBMS: MySQL, PostgreSQL, Oracle, SQL Server

### 특징
- 행과 열로 구성된 테이블에 데이터 저장
- 테이블 사이의 관계를 `JOIN`으로 처리
- 미리 정의된 스키마 사용
- SQL이라는 표준 질의 언어 사용
- 트랜잭션과 데이터 일관성을 중요하게 처리

```sql
SELECT *
FROM 회원
WHERE 회원ID = 1;
```

## 2. NoSQL
- Not Only SQL이라는 의미
- 관계형 테이블 구조를 반드시 사용하지 않는 데이터베이스
- 대표적인 DBMS: MongoDB, Redis, Cassandra, Neo4j

### 특징
- 고정된 테이블 구조가 없어 비교적 유연함
- 대량의 비정형 데이터를 저장하기 편리함
- 여러 서버로 데이터를 분산하기 쉬움
- 데이터 구조에 따라 여러 유형으로 나뉨
- 복잡한 관계나 트랜잭션 처리에서는 SQL보다 불편할 수 있음

```JSON
{
    "userId": 1,
    "name": "정훈",
    "skills": ["Java", "Python", "SQL"]
}
```

### NoSQL의 종류

#### Key-Value
- 키를 이용하여 값을 저장함

```text
"user:1" -> 사용자 정보
```

- 대표 DB: Redis
- 활용: 캐시, 세션 저장

#### Document
- JSON과 비슷한 문서 형태로 저장함
- 대표 DB: MongoDB
- 활용: 상품 정보, 게시글, 사용자 프로필

#### Column Family
- 데이터를 열 중심으로 저장하고 분산 처리에 적합
- 대표 DB: Cassandra, HBase
- 활용: 대규모 로그, 시계열 데이터

#### Graph
- 노드와 간선으로 데이터의 관계를 표현함
- 대표 DB: Neo4j
- 활용: 소셜 네트워크, 추천 시스템, 경로 탐색

## 3. 주요 차이점

| 구분      | SQL           | NoSQL                 |
| ------- | ------------- | --------------------- |
| 데이터 구조  | 테이블           | 문서, 키-값, 그래프 등        |
| 스키마     | 고정적           | 유연함                   |
| 관계 처리   | JOIN 사용       | 중첩 저장 또는 애플리케이션에서 처리  |
| 확장 방식   | 주로 수직 확장      | 주로 수평 확장              |
| 데이터 일관성 | 강한 일관성 중시     | 가용성과 확장성을 중시하는 경우가 많음 |
| 트랜잭션    | 복잡한 트랜잭션에 적합  | 제품과 유형에 따라 지원 수준이 다름  |
| 적합한 데이터 | 정형 데이터        | 비정형·반정형 데이터           |
| 대표 사례   | 금융, 주문, 회원 관리 | 캐시, 로그, SNS, 실시간 데이터  |


## 4. 확장 방식

### 수직 확장
- 한 서버의 CPU, 메모리 등의 성능을 높이는 방식

```text
기존 서버 -> 더 성능이 좋은 서버
```

- SQL 데이터베이스에서 전통적으로 많이 사용

### 수평 확장
- 서버의 개수를 늘려 분산하는 방식

```text
서버 1대 -> 서버 여러 대
```

- NoSQL은 일반적으로 분산 저장과 수평 확장이 쉽도록 설계됨
- SQL도 샤딩, 복제, 분산 SQL 등을 통해 수평 확장이 가능함

## 5. 어떤 것을 선택하는가?

### SQL이 적합한 경우
- 데이터 구조가 명확한 경우
- 데이터 간 관계가 복잡한 경우
- JOIN이 자주 필요한 경우
- 결제나 주문처럼 정확한 트랜잭션이 중요한 경우

### NoSQL이 적합한 경우
- 데이터 구조가 자주 변경되는 경우
- 대량의 데이터를 여러 서버에 분산해야 하는 경우
- 빠른 읽기•쓰기가 중요한 경우
- 캐시, 로그, 실시간 데이터 등을 처리하는 경우

## 참고

### 면접 답변 예시
```text
SQL 데이터베이스는 데이터를 테이블 형태로 저장하고, 스키마와 관계를 기반으로 데이터의 일관성과 트랜잭션을 중요하게 처리합니다.
반면 NoSQL은 문서, 키-값, 그래프 등 다양한 형태로 데이터를 저장하며, 유연한 구조와 수평 확장에 유리합니다. 
따라서 복잡한 관계와 트랜잭션이 중요하면 SQL을, 대규모 분산 처리와 유연한 데이터 구조가 중요하면 NoSQL을 선택할 수 있습니다.
```
